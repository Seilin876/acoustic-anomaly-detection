## Fixes applied during this run

Two issues were observed and patched in `anomaly_detection.py`:

1. **PyArrow schema coercion on dirty object columns** (carried over from the
   `error code` patch the user supplied). Object-dtype columns are cast to
   string before `to_parquet`, so mixed-type cells (numbers and strings in the
   same column) no longer abort the conversion.

2. **`RuntimeWarning` spam from `sklearn.decomposition._base`**
   (`divide by zero` / `overflow` / `invalid value encountered in matmul`).
   Investigated:
   - Max absolute scaled value across all groups: **~64**, well within float64
     range. Not a real overflow.
   - 33 groups triggered the warnings even with `np.nan_to_num` guarding the
     scaler outputs.
   - Root cause is a known sklearn/numpy interaction in PCA's internal SVD path
     where FP error flags fire on benign computations.
   - **Fix:** keep the defensive `nan_to_num` (cheap insurance) and
     `warnings.filterwarnings("ignore", category=RuntimeWarning,
     module=r"sklearn\.decomposition\._base")`.
   - Anomaly count is identical before and after the fix (3,268), confirming
     the warnings were noise, not silent corruption.

## Reproduce

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 anomaly_detection.py
```

Wall-clock for this run: roughly 30–40 s for the two days combined,
dominated by `pd.read_excel` on the first invocation. Re-runs are sub-second
for the Parquet load.

## Observations / follow-ups

- 26 groups failed the 300-sample threshold and were dropped — worth a
  separate inspection so we don't silently miss low-volume stations.
- Score ceiling at ~-5e-6 (just barely anomalous) suggests `contamination=0.05`
  is producing some borderline labels; consider switching to a fixed
  `Anomaly_Score` threshold once we have ground truth, instead of a fixed
  contamination fraction.
- Groups with a single dominant PCA component (11 out of 36) are candidates
  for a much simpler univariate check — PCA + Isolation Forest may be
  overkill there.
